# cashflow

Веб-сервис для учёта движения денежных средств (ДДС). Можно создавать, редактировать
и удалять записи о поступлениях/списаниях, фильтровать их по дате, статусу, типу,
категории и подкатегории, а также управлять самими справочниками.

Сделано на Django + DRF, база — SQLite, фронтенд — обычные Django-шаблоны с Bootstrap
и немного JS для выпадающих списков.

## Как запустить

1. Создать и активировать виртуальное окружение:

```bash
python3 -m venv venv
source venv/bin/activate
```

2. Поставить зависимости:

```bash
pip install -r requirements.txt
```

3. Накатить миграции (создастся db.sqlite3, отдельно настраивать БД не нужно):

```bash
python manage.py migrate
```

При миграции сразу подтянутся стартовые справочники: статусы (Бизнес, Личное,
Налог), типы (Пополнение, Списание) и несколько примеров категорий/подкатегорий
из ТЗ (Инфраструктура -> VPS/Proxy, Маркетинг -> Farpost/Avito).

4. Если нужен доступ в стандартную админку Django - создать суперпользователя:

```bash
python manage.py createsuperuser
```

5. Запустить сервер:

```bash
python manage.py runserver
```

Дальше открываем http://127.0.0.1:8000/

## Что где

- `/` — список записей + фильтры
- `/records/create/` — новая запись
- `/records/<id>/edit/` — редактирование
- `/references/` — справочники (статусы, типы, категории, подкатегории)
- `/admin/` — админка Django
- `/api/` — REST API (записи и все справочники)

## Про бизнес-правила

Категория привязана к типу, подкатегория к категории. В форме выпадающие списки
подгружаются по цепочке через API (тип -> категория -> подкатегория), так что
несовместимую комбинацию выбрать не получится. То же самое проверяется и на
сервере, так что даже если обойти интерфейс — сохранить некорректную запись не
выйдет.

Сумму храню в копейках (целое число), чтобы не было проблем с округлением, а
показываю уже в рублях.

Справочник, на который ссылаются существующие записи, удалить нельзя — вместо
удаления появится сообщение об ошибке.

## Прочее

Всё работает из коробки на SQLite и DEBUG=True. Для продакшна нужно поменять
SECRET_KEY, выставить DEBUG=False и заполнить ALLOWED_HOSTS в
cashflow/settings.py.
